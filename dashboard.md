# Dashboard Page Overrides

> 项目：慢病用药小管家
>
> 页面类型：Patient Medication Dashboard / 慢病用药工作台
>
> 本文件覆盖 `design-system/慢病用药小管家/MASTER.md` 中不适合 Dashboard 的规则。

## 1. Page Purpose

Dashboard 不是 landing page，也不是宣传页。

它的任务是帮助用户完成今天的慢病管理动作：

- 今天要吃什么药。
- 哪个药快不够了。
- 什么时候复诊。
- 复诊前还缺什么资料。
- 家属是否需要知道。

## 2. First Screen Priority

桌面端首屏必须包含：

1. 今日最高优先级任务。
2. 今日用药计划摘要。
3. 低库存/复诊临近状态。
4. 明确 CTA。

移动端首屏必须包含：

1. 今日最高优先级任务。
2. 下一次待服药。
3. 打卡按钮。

## 3. Layout Override

推荐桌面布局：

- 左侧：固定导航与患者摘要。
- 中间：今日任务、用药计划、复诊路径。
- 右侧：AI 建议、库存、家属同步。

推荐移动布局：

- 顶部：品牌 + 当前患者。
- 主体：单列任务流。
- 底部：可选固定行动区。

最大内容宽度：

- 不强制居中 1200px。
- 允许应用工作台使用全宽，但内容区需要有 24-32px 页面边距。

## 4. Components Required

### 今日最高优先级卡

必须位于首屏最前。

示例：

“二甲双胍将在复诊前耗尽”

CTA：

- 主按钮：预约复诊。
- 次按钮：查看补药清单。

### 下一次服药卡

移动端优先展示。

字段：

- 时间。
- 药名。
- 剂量。
- 服用场景。
- 打卡按钮。

### 药品库存模块

按剩余天数升序排列。

低库存药品必须有明确视觉优先级和行动入口。

### 复诊续方路径

使用垂直时间线。

当前阻塞项必须具体，例如：

“缺少 1 次晚间血压记录。”

### AI 建议面板

不做黑色科技感侧栏。

推荐浅色或极浅绿底，使用清楚的标题和行动建议。

## 5. Color Override

Dashboard 禁用视频深色 overlay 策略。

使用浅背景：

- 页面背景：`#F6FAF8`
- 卡片背景：`#FFFFFF`
- 风险卡背景：根据风险使用浅色底
- 主 CTA：`#0F7A5A`

## 6. Content Rules

页面内不出现营销式文案：

- “提升效率 86%”
- “下一代 AI 医疗”
- “重新定义慢病管理”

推荐使用行动式文案：

- “今晚 19:00 服用二甲双胍”
- “二甲双胍剩余 9 天”
- “复诊前还需 1 次晚间血压”
- “已为医生整理近 30 天摘要”

## 7. Density Rules

- 桌面端可以中等信息密度。
- 移动端必须低密度。
- 关键任务之间保持明显间距。
- 不把 4 个以上状态塞进同一张卡。

## 8. Dashboard Acceptance Checklist

- [ ] 首屏不是 hero landing page。
- [ ] 首屏有明确今日任务。
- [ ] 用药打卡入口可见。
- [ ] 低库存药品可见。
- [ ] 复诊续方路径可见或有入口。
- [ ] AI 建议是行动建议，不是诊断结论。
- [ ] 移动端不需要缩放即可阅读。
- [ ] 每个风险都有下一步按钮。
