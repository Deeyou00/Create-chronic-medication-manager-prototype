# 慢病用药小管家

一个面向慢病患者与家属的静态网页原型，聚焦线上复诊后的连续照护体验：按时吃药、库存预警、复诊续方、家属提醒、医生摘要。

## 项目简介

这个原型不是医生后台，也不是单纯闹钟，而是一个站在患者视角设计的“家庭用药工作台”。

它解决的是线上复诊后的真实断点：

- 忘记按时服药。
- 不清楚药还能吃几天。
- 复诊前资料不齐，容易来回补交。
- 家属想帮忙，但不知道该在什么时候介入。
- 家里明明还有药，却重复购买。
- 复诊时说不清最近的服药和指标变化。

## 本次原型重点

- 首屏聚焦“今天最重要的事”，降低中老年用户的决策负担。
- 把“补资料 -> 预约复诊 -> 核对处方 -> 按需购药”做成连续路径。
- 每个风险都对应明确下一步，而不是只做提醒。
- 强化信任感：不会自动改药，不会自动把摘要发给医生，家属也看不到隐私备注。
- 用前端状态模拟完整闭环，而不是只做静态展示。

## 功能亮点

- 首页：今日重点、下一次吃药、今日待办。
- 用药计划：支持打卡、补记、未服用记录。
- 药品库存：按剩余天数排序，提醒低库存与避免重复购药。
- 复诊续方：支持补晚间血压、推进预约状态、更新时间线。
- 家属管理：支持新增家属、权限编辑、暂停/恢复提醒。
- 给医生的摘要：根据服药与指标状态动态更新。
- 新增计划：家属可代填，患者确认后才生效。
- 可访问性：大字模式、键盘可用、状态有文本反馈。

## 交互说明

这个仓库当前是纯静态原型，没有后端。

为了让面试官更容易体验，我在页面里加入了可操作的前端状态模拟：

- 点击“我已吃药”会更新首页、摘要和任务状态。
- 录入晚间血压后，复诊状态会从“还差一步”变成“可预约”。
- 点击预约复诊后，时间线和首页重点会继续推进。
- 家属提醒可以暂停/恢复，并即时反映到列表状态。
- 表单包含必填、格式校验、保存中和成功反馈。

## 如何运行

方式一：直接打开

双击 `index.html` 即可在浏览器中查看。

方式二：本地静态服务

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://127.0.0.1:8000/index.html
```

## 推荐面试官查看路径

1. 首页：先看“今日重点”和“下一次吃药”。
2. 复诊续方：录入一次晚间血压，观察状态如何推进。
3. 家属管理：暂停一个家属提醒，确认权限和信任边界。
4. 给医生的摘要：查看服药、指标、复诊状态如何动态联动。
5. 新增计划：体验家属代填后由患者确认的流程。

## 项目结构

- `index.html`: 主原型页面，包含 HTML / CSS / JavaScript。
- `design-system.md`: 设计系统说明。
- `design-system/`: 页面级设计规范与主设计原则。
- `skills-lock.json`: 本地技能锁文件。

## 设计与产品思路

- 目标用户：中老年慢病患者、患者家属、慢病管理医生或药师。
- 风格方向：温和、可信赖、有秩序、低学习成本。
- 产品边界：只做提醒、记录和整理，不代替医生诊断或改药。
- 设计原则：首页只放最重要的信息，避免信息堆叠和后台感。

## 说明

- 本项目为面试展示用途的静态原型。
- 未接入真实医疗数据、订单系统或消息服务。
- 所有数据均为演示内容。
