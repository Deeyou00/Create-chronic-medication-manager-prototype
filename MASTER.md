# 慢病用药小管家 Design System Master

> 使用规则：构建具体页面时，先读取 `design-system/慢病用药小管家/pages/[page].md`。如果页面文件存在，以页面规则覆盖本文件；否则严格遵循本 Master。

生成来源：`ui-ux-pro-max`

项目类型：Healthcare App / 慢病用药管理 / 复诊续方助手

核心定位：面向慢病患者与家属的家庭用药与续方管家，连接“问诊、购药、用药、指标记录、续方复诊”的连续照护链路。

## 1. Product Principles

### 设计目标

- 让患者 5 秒内知道今天最该做什么。
- 让患者清楚知道“什么时候吃药、药还够几天、什么时候复诊”。
- 让家属快速判断是否需要介入。
- 让复诊前的用药、库存、指标摘要更容易被医生理解。

### 设计语气

- 可信赖
- 清晰
- 温和
- 有秩序
- 少焦虑
- 家庭照护感
- 医疗级但不冰冷
- 面向中老年友好

### 明确避免

- 不做视频落地页。
- 不做数据大屏。
- 不做深色科技驾驶舱。
- 不做复杂电商首页。
- 不用 AI 紫粉渐变。
- 不用高刺激警报式视觉。

## 2. Visual Direction

视觉方向：温和医疗工作台。

界面应该像一张可靠、干净、清楚的家庭健康计划表。它可以有产品质感，但必须优先服务可读性和行动效率。

记忆点：

- 首屏只突出今日最重要风险。
- 用“连续照护链路”表达产品差异。
- 用明确的状态色帮助用户处理漏服、库存不足、复诊临近。
- 每个风险都要绑定一个下一步动作。

## 3. Color Tokens

`ui-ux-pro-max` 推荐的基础方向是 Calm cyan + health green。结合慢病场景，保留青绿色医疗感，但降低饱和度，避免页面像科技 SaaS。

| Token | Hex | Usage |
| --- | --- | --- |
| `--color-primary` | `#0F7A5A` | 主品牌色、完成状态、当前步骤 |
| `--color-primary-strong` | `#075D46` | 主按钮 hover、重点标题 |
| `--color-cyan` | `#0891B2` | 医生、指标、信息同步状态 |
| `--color-cyan-soft` | `#E6FAFD` | 轻信息背景 |
| `--color-success-soft` | `#EAF7F1` | 已完成、正常状态背景 |
| `--color-warning` | `#B7791F` | 待处理、复诊临近、库存偏低 |
| `--color-warning-soft` | `#FFF3D7` | 轻风险背景 |
| `--color-danger` | `#C2413D` | 断药风险、异常指标、高优先级风险 |
| `--color-danger-soft` | `#FFE8E5` | 高风险背景 |
| `--color-background` | `#F6FAF8` | 页面背景 |
| `--color-surface` | `#FFFFFF` | 卡片、表单、面板 |
| `--color-surface-soft` | `#F1F8F5` | 次级容器背景 |
| `--color-text` | `#16332B` | 主文本 |
| `--color-muted` | `#64716D` | 辅助文本 |
| `--color-border` | `#DCE7E2` | 分割线、输入框边框 |

### 色彩规则

- 页面以浅色为主，降低中老年用户视觉压力。
- 绿色表达“可继续、已完成、健康稳定”。
- 琥珀色表达“需要注意但不紧急”。
- 红色只用于真正影响连续用药的风险。
- 不用红色作为装饰色。
- 文本对比度最低满足 WCAG AA，关键任务区域尽量接近 AAA。

## 4. Typography

`ui-ux-pro-max` 推荐：Heading 使用 Figtree，Body 使用 Noto Sans。

实际落地建议：

- 中文主字体：`Noto Sans SC`, `PingFang SC`, `Microsoft YaHei`, sans-serif
- 英文和数字标题：`Figtree`, `Noto Sans SC`, sans-serif
- 指标数字：可使用 `DIN Alternate` 或系统等宽数字

### 字号层级

| Role | Desktop | Mobile | Notes |
| --- | ---: | ---: | --- |
| Page title | 32-40px | 24-28px | 只用于页面最高标题 |
| Section title | 20-24px | 18-20px | 模块标题 |
| Card title | 16-18px | 16px | 药名、任务标题 |
| Body | 15-16px | 15-16px | 面向中老年，不低于 15px |
| Helper text | 13-14px | 13px | 辅助说明 |
| Metric number | 32-44px | 28-36px | 依从率、库存天数、血压 |
| Badge | 12-13px | 12px | 状态标签 |

### 排版规则

- 中文段落行高使用 `1.55-1.75`。
- 长说明尽量控制在两行以内。
- 不使用负字距。
- 不随 viewport 宽度线性缩放字体。
- 所有关键数字应使用 tabular numbers。

## 5. Layout

### 页面结构

推荐主结构：

1. 今日最重要任务
2. 今日用药计划
3. 关键健康指标
4. 药品库存
5. 复诊续方路径
6. AI 复诊摘要
7. 家属/医生同步设置

### 栅格

| Viewport | Layout |
| --- | --- |
| 375-767px | 单列，优先展示今日任务和用药计划 |
| 768-1199px | 两列或主内容 + 侧边信息 |
| 1200px+ | 左导航 + 主工作区 + 右侧辅助信息 |

### 间距 Tokens

| Token | Value | Usage |
| --- | --- | --- |
| `--space-1` | `4px` | 紧密文本间距 |
| `--space-2` | `8px` | 图标和文字间距 |
| `--space-3` | `12px` | 标签、表单内部 |
| `--space-4` | `16px` | 卡片内边距 |
| `--space-5` | `20px` | 模块间距 |
| `--space-6` | `24px` | 页面边距 |
| `--space-8` | `32px` | 大区块间距 |

### 圆角与阴影

- 组件圆角：8px。
- 大容器圆角：8px 或 10px。
- 不使用过大的 20px+ 圆角。
- 阴影只用于层级，不用于装饰。

```css
:root {
  --radius-sm: 6px;
  --radius-md: 8px;
  --radius-lg: 10px;
  --shadow-sm: 0 1px 2px rgba(20, 43, 37, 0.05);
  --shadow-md: 0 8px 24px rgba(20, 43, 37, 0.08);
  --shadow-lg: 0 18px 45px rgba(20, 43, 37, 0.1);
}
```

## 6. Core Components

### A. 今日任务卡

用途：首屏最高优先级行动。

内容：

- 风险类型：漏服、低库存、复诊临近、指标待补。
- 简短原因。
- 推荐行动。
- 主按钮 + 次按钮。

状态：

- Normal：绿色。
- Attention：琥珀色。
- Critical：红色。

规则：

- 只展示 1 个最高优先级任务。
- 不堆叠多个警报。
- 必须有明确 CTA。

### B. 用药计划行

内容：

- 时间。
- 药品名称。
- 剂量。
- 服用场景。
- 库存剩余。
- 状态标签。
- 打卡按钮。

状态：

- 已服用：绿色标签 + “已打卡”。
- 待服用：琥珀标签 + “打卡”。
- 漏服：红色标签 + “补记/联系医生”。
- 暂停服用：灰色标签 + 医嘱说明。

### C. 药品库存条

规则：

- `< 10 天`：红色，高优先级。
- `10-20 天`：琥珀色，提醒。
- `> 20 天`：绿色，正常。

库存条必须展示可用天数，不只展示百分比。

### D. 复诊续方路径

节点：

1. 补齐复诊资料。
2. 发起线上复诊。
3. 处方核对。
4. 按需购药。
5. 新计划生效。

规则：

- 使用时间线，不使用复杂流程图。
- 当前节点高亮。
- 阻塞项必须写清楚，例如“缺少 1 次晚间血压”。

### E. AI 建议面板

用途：把系统判断翻译成用户能执行的建议。

文案结构：

1. 发现了什么。
2. 为什么重要。
3. 今天建议做什么。

禁用表达：

- AI 诊断。
- 系统判定严重。
- 必须马上。
- 危险。

推荐表达：

- 建议补齐。
- 即将不足。
- 可提前预约。
- 已为复诊整理。
- 需要医生确认。

### F. 家属同步控件

默认开启：

- 漏服提醒。
- 低库存提醒。
- 复诊前提醒。

交互规则：

- 开关旁必须解释通知对象和触发条件。
- 不隐藏在深层设置里。

## 7. Interaction Rules

- 所有可点击元素至少 44x44px。
- 所有按钮有 hover、active、focus-visible 状态。
- 操作后使用 toast 或 inline feedback。
- 不用动画表达医学风险。
- 支持 `prefers-reduced-motion`。
- 键盘可访问。
- 表单错误不能只靠颜色表达。

## 8. Accessibility Rules

来自 `ui-ux-pro-max` 的核心要求：

- 大文本与高对比。
- 明确 focus ring，建议 3-4px。
- 语义化 HTML。
- 屏幕阅读器可理解。
- 移动端触控目标不小于 44px。

落地规则：

- 主文本对比度不低于 4.5:1。
- 状态标签必须包含文字，不能只靠颜色。
- 风险提示使用 `role="status"` 或 `aria-live`。
- 图标使用 Lucide/Heroicons 等一致图标集。
- 不使用 emoji 作为功能图标。

## 9. Motion

允许：

- 150-240ms 的 hover/focus 过渡。
- 轻量 toast 出现。
- 卡片进入时轻微淡入。

避免：

- 大幅度弹跳。
- 自动循环动画。
- 大面积背景动效。
- 可能分散中老年用户注意力的装饰动画。

## 10. Mobile Rules

移动端首屏顺序：

1. 今日最重要任务。
2. 下一次待服药。
3. 今日完成度。
4. 低库存药品。

规则：

- 单列布局。
- 底部可保留核心行动按钮。
- 用药打卡按钮必须全宽或足够大。
- 导航最多 4 个一级入口。
- 避免横向滚动，导航标签除外。

## 11. Pre-Delivery Checklist

- [ ] 首屏是否 5 秒内能理解今日重点？
- [ ] 每个风险是否都有明确下一步？
- [ ] 低库存、漏服、复诊临近是否状态清楚？
- [ ] 是否避免了营销落地页结构？
- [ ] 是否避免了医生后台/数据大屏感？
- [ ] 是否所有点击目标不小于 44px？
- [ ] 是否有清晰 focus-visible？
- [ ] 是否支持 375px、768px、1024px、1440px？
- [ ] 是否没有纯颜色状态表达？
- [ ] 是否没有 emoji 功能图标？
